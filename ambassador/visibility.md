# Making Invisible Work Visible Cheat Sheet

If no one sees it, it doesn't count for your career.

## What Is Invisible Work?

| Type           | Examples                                                     |
| -------------- | ------------------------------------------------------------ |
| **Glue work**  | Onboarding, documentation, process improvement, filling gaps |
| **Unlocking**  | Unblocking teammates, answering questions, reviewing PRs     |
| **Enabling**   | Building tools, creating templates, improving DX             |
| **Preventing** | Catching bugs before prod, heading off bad decisions         |
| **Connecting** | Cross-team coordination, relationship maintenance            |

**Tanya Reilly's definition** (from "Being Glue"): Work that makes the whole
team better but doesn't look like "engineering."

## Why It Goes Unrecognized

| Factor                | Mechanism                                            |
| --------------------- | ---------------------------------------------------- |
| **No artifacts**      | Nothing to point to in a promo packet                |
| **Attribution gap**   | "The project succeeded" — who enabled that?          |
| **Not technical**     | Promo rubrics reward technical depth                 |
| **Success = absence** | The outage that didn't happen has no postmortem      |
| **Diffuse impact**    | Helping 10 people a little vs. one big visible thing |
| **Gendered patterns** | Research shows women do more "non-promotable tasks"  |

**Research note**: Babcock et al. (2017) found women are 48% more likely to
volunteer for non-promotable tasks and receive these requests more often. The
work is real; the credit is not.

## The Glue Work Trap

When helping stalls your career.

### Signs You're Trapped

- You're the go-to person for questions but not for projects
- Your promo packet is thin despite being "essential"
- You feel resentful about helping
- You've been "almost ready" for promotion for 2+ cycles
- Leadership doesn't know your name

### Why It Happens

- Helping feels good; coding on a visible project is less immediately rewarding
- Teams _need_ this work done
- No one else steps up
- You're told you're "so valuable" (but not promoted)

### How to Escape

- Track where your time goes for 2 weeks
- Identify which glue work is strategic vs. accidental
- Delegate or stop the low-value invisible work
- Shift to visible work, even if it feels selfish
- Have explicit conversation with manager about promo criteria

## Strategic vs Accidental Glue Work

| Strategic Glue                   | Accidental Glue                 |
| -------------------------------- | ------------------------------- |
| Aligns with your growth goals    | Just filling gaps               |
| Creates reusable artifacts       | One-off help                    |
| Builds visibility and reputation | Known only to the person helped |
| You can cite it in promo packet  | "I helped the team" (vague)     |
| You chose it                     | You got stuck with it           |

**The test**: Does this help me grow, or am I just preventing things from
falling apart?

## Creating Artifacts from Invisible Work

Transform invisible work into visible evidence.

| Invisible Work          | Artifact                                        |
| ----------------------- | ----------------------------------------------- |
| Mentoring engineers     | Testimonial from mentee; their successful promo |
| Reviewing PRs           | Summary of patterns caught, bugs prevented      |
| Cross-team coordination | Decision document, project charter              |
| Answering questions     | FAQ doc, runbook, onboarding guide              |
| Improving process       | Metrics showing improvement (cycle time, bugs)  |
| Unblocking teammates    | Shout-outs collected; impact on their projects  |

## The Brag Document

A running log of your accomplishments. You'll forget; the doc won't.

### Structure

```markdown
# Brag Document — [Year]

## Q1

### Project: [Name]

- What I did: [Specific contribution]
- Impact: [Quantified where possible]
- Stakeholders: [Who knows about this]

### Glue Work

- [Contribution]: [Impact]
- [Who benefited]: [Their testimonial or outcome]

## Feedback Received

- [Quote from peer/manager]
```

### Cadence

- **Weekly**: 5-minute dump of what you did
- **Monthly**: Organize and add impact/metrics
- **Quarterly**: Review with manager; refine for promo packet
- **Yearly**: Synthesize for performance review

### How to Use It

- In 1:1s: "Here's what I've been working on that you might not see"
- For promo: Source material for packet
- For self-assessment: Counter impostor syndrome with evidence

## Framing Language

How you describe work matters for visibility.

| Weak Framing                  | Strong Framing                                                                     |
| ----------------------------- | ---------------------------------------------------------------------------------- |
| "I helped with the project"   | "I designed the caching layer that reduced latency by 40%"                         |
| "I answered questions"        | "I created onboarding docs that reduced ramp time from 4 weeks to 2"               |
| "I did code reviews"          | "I caught a security vulnerability in 3 PRs that would have exposed customer data" |
| "I coordinated between teams" | "I aligned Platform and Product on the API contract, unblocking 2 teams"           |
| "I helped people"             | "I enabled [Person] to ship [Project] by [specific action]"                        |

**The formula**: "I [action] which resulted in [measurable outcome] for
[stakeholder]."

## 1:1 → 1:Many → Systemic

Scaling your help increases impact and visibility.

| Level        | Example                                  | Visibility      |
| ------------ | ---------------------------------------- | --------------- |
| **1:1**      | Answer a colleague's question            | Only they know  |
| **1:many**   | Write docs so 50 people don't ask        | Discoverable    |
| **Systemic** | Build tooling that prevents the question | Measured, cited |

**The shift**: Each time you help someone, ask: "Can I create something that
helps everyone?"

## Gender Dynamics

Research consistently shows:

- Women volunteer more for non-promotable tasks (Babcock et al.)
- Managers ask women more often to do "office housework"
- Men receive more credit for the same visible work
- Glue work is often coded as "not technical enough"

### What to Do About It

- Track your task distribution
- Say no to non-promotable tasks at the same rate as male peers
- Make your contributions explicit ("I'd be happy to take notes, but I did it
  last three times")
- Advocate for promo criteria that recognize glue work
- If you're a manager: distribute housework equitably; track it

## When to Stop (Temporarily)

Sometimes the answer is to stop doing invisible work.

### Signs It's Time

- You're resentful
- It's not valued in your promo criteria
- Someone else could do it
- It's preventing you from doing visible work
- You've been doing it for 6+ months with no recognition

### How to Stop Gracefully

- Document what you do so others can pick it up
- Train a replacement
- Let things break (sometimes necessary for the work to become visible)
- Have explicit conversation: "I need to focus on X for my growth. Can we
  redistribute Y?"

**The uncomfortable truth**: Sometimes the only way to make invisible work
visible is to stop doing it and let people notice the gap.
